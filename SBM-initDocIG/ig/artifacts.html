Structures: Logical Models

These define data models that represent the domain covered by this implementation guide in more business-friendly terms than the underlying FHIR resources.

PN: PersonName (V3 Data Type)

A name for a person. A sequence of name parts, such as given name or family name, prefix, suffix, etc. Examples for person name values are “Jim Bob Walton, Jr.”, “Adam Everyman”, etc. A person name may be as simple as a character string or may consist of several person name parts, such as, “Jim”, “Bob”, “Walton”, and “Jr.”. PN differs from EN because the qualifier type cannot include LS (Legal Status).
PatientRole

Description du patient/usager
Person

Person permet de décrire : une personne physique
RelatedEntity

RelatedEntity permet de décrire : un informateur non professionnel / personne de confiance / personne à prévenir en cas d’urgence / aidant / personne aidée.
assignedAuthor

assignedAuthor contient les éléments permettant de décrire l’auteur.
assignedCustodian

assignedCustodian, contient l’élément representedCustodianOrganization caractérisant la structure conservant le document.
assignedEntity

AssignedEntity est utilisé dans les éléments suivants : dataEnterer, informant, legalAuthenticator, authenticator, performer, responsibleParty et encounterParticipant.
associatedEntity

AssociatedEntity représente les caractéristiques du professionnel et/ou de l’établissement participant.
authenticator

Authenticator, professionnel (personne physique) attestant la validité du contenu du document.
author

author permet d’enregistrer un auteur du document.
authorization

Authorization, permet de documenter qu’un consentement éclairé a été obtenu et d’indiquer quel type de consentement a été fourni.
componentOf

componentOf permet d’associer le document à une prise en charge du patient/usager.
custodian

Structure chargée de la conservation du document.
dataEnterer

DataEnterer contient les informations relatives à l’opérateur de saisie de tout ou partie du contenu du document.
documentationOf

DocumentationOf représente un évènement (acte, traitement, diagnostic, etc…) décrit dans le document. Il y a au minimum une occurrence de cet élément pour décrire l’évènement principal avec obligatoirement une date de début et un exécutant.
inFulfillmentOf

​InFulfillmentOf permet d’Associer un document à une prescription .
informant

Informateur / personne de confiance / personne à prévenir en cas d’urgence / aidant / personne aidée.
informationRecipient

InformationRecipient permet d’enregistrer une personne déclarée comme destinataire prévu du document.
legalAuthenticator

LegalAuthenticator représente les caractéristiques du professionnel et/ou de l’établissement participant.
participant

participant représente toute personne/structure impliquée dans les évènements décrits par le document qui n’a pas été mentionné ailleurs.
patient

Personne physique
performer

Performer est l’exécutant de l’évènement documenté. Il est obligatoire et son attribut nullFlavor interdit pour l’évènement documenté principal.
recordTarget

Patient/Usager concerné par le document.
relatedDocument

RelatedDocument référence un document existant (à remplacer ou transformé).
serviceEvent

ServiceEvent représente un évènement (acte, traitement, diagnostic, etc…) décrit dans le document. L’occurrence de documentationOf/serviceEvent contenant les données de l’évènement documenté principal doit inclure un élément effectiveTime et un élément performer renseignés, sans recours à l’attribut nullFlavor.
